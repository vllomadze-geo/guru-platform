# ГУРУ, MVP платформы v0.27

## Что изменено в v0.27

Пересобран блок **Gate 1 → Юнит-экономика: целевые CPA/DRR, AOV/LTV, маржинальность**.

Теперь это один маршрут принятия решения о запуске рекламы:

`Чек → маржа → LTV → DRR → CPA/CPL → решение о запуске`.

## Реализовано

1. Убрана логика разрозненных карточек CPA, DRR, AOV, LTV, маржинальности и ограничений.
2. Добавлен один маршрут из 6 accordion-разделов:
   - Выручка / AOV;
   - Маржинальность;
   - LTV;
   - Допустимый DRR;
   - Целевой CPA / CPL;
   - Ограничения по экономике.
3. Вверху блока добавлены: продукт / услуга, период оценки, модель продажи, финальный допустимый CPA / CPL.
4. Добавлен автоматический итог: допустимый CPA, допустимый CPL, допустимый DRR, решение, что можно и нельзя продвигать.
5. Добавлены формулы:
   - Допустимый CPA = AOV × маржа × допустимая доля на рекламу;
   - Допустимый CPL = CPA × конверсия из лида в продажу.
6. Статус блока рассчитывается автоматически.
7. Новые окна и отдельные разделы не добавлялись.

## Принцип

Юнит-экономика должна не собирать финансовые поля, а давать ответ: сколько можно платить за заявку / клиента и можно ли запускать рекламу.
