# ГУРУ, v1.1.1.1-clean Gate 0

## Назначение версии

Точечное обновление только одного блока:

`Gate 0 / Паспорт проекта → Продукт, сегмент и задача клиента`

Главный принцип: не ломать уже подключённые возможности, не убирать Яндекс, не делать массовый рефакторинг, двигаться блок за блоком.

## Что оставлено

- Supabase-подключение из v1.0.
- Проверка подключения Supabase.
- Яндекс-блоки и Яндекс-интеграция в интерфейсе.
- Существующая структура Gate 1–7.
- Существующие файлы, стили, seed-data и API-проверка.

## Что изменено

- Блок `Продукт, сегмент и задача клиента` возвращён к простой паспортной структуре.
- Внутри блока оставлены три ключевых поля:
  - `Что продаём`
  - `Кому продаём`
  - `Ради какого результата`
- Старый смысл инструкции возвращён:
  - Инструменты: бриф, интервью, CRM, сайт, отдел продаж.
  - Блок отвечает за понимание: что продаётся, кому продаётся и какую задачу клиента решает.
  - На выходе должно быть понятно: что продаём, кому и ради какого результата.
- Три поля сохраняются как глобальные данные проекта:
  - `project.whatSell`
  - `project.targetSegment`
  - `project.clientResult`
- Эти же значения синхронизируются со структурированным доказательством блока.
- Добавлены маленькие индикаторы: `Сохранено в проекте`, `Используется в Gate 1 / Gate 3 / Gate 4`.

## Что убрано из этого блока

- `Контекст 1 из 10`
- `Стратегическое решение`
- `Источник решения`
- `Следующий шаг`
- счётчики символов
- большие textarea
- сложная GPT-форма
- лишняя сетка 2×2

## Что не трогалось

- Supabase-таблицы.
- Vercel Environment Variables.
- Яндекс.
- Gate 1–7, кроме логической передачи глобальных значений из Gate 0.
