# ГУРУ, MVP платформы v0.31

## Что изменено в v0.31

Пересобран **Gate 6 → Маркетинговый календарь событий**.

Блок больше не работает как таблица дат. Теперь это маршрут принятия решений:

`событие → спрос → сегмент → офер → канал → дедлайн → запуск`.

## Реализовано

1. Вверху Gate 6 добавлен единый автоконтекст проекта:
   - период;
   - проект;
   - главная аудитория;
   - каналы;
   - автоматический статус.
2. Календарь разделён на 7 сезонных accordion-разделов:
   - Зима;
   - Ранняя весна;
   - Весна → Лето;
   - Лето;
   - Осень;
   - Поздняя осень → Зима;
   - Подвижные события.
3. Каждый сезон показывает:
   - фазу сезона;
   - поведенческие факторы;
   - что продаётся на самом деле;
   - компактные карточки событий.
4. В карточке события оставлена рабочая логика:
   - актуальность;
   - сценарий спроса;
   - сегмент;
   - офер;
   - канал;
   - материалы / дедлайн;
   - готовность материалов.
5. Внизу Gate 6 автоматически формируется финальный план кампаний.
6. Статусы событий считаются автоматически:
   - Не начато;
   - В работе;
   - Готово к подготовке;
   - Готово к запуску;
   - Проблема;
   - Исключено.
7. Убран основной режим простой таблицы дат.
8. Повторяющийся автоконтекст больше не дублируется внутри каждого события.

## Принцип

Маркетинговый календарь не нужен ради календаря. Он нужен, чтобы выбрать релевантный повод, связать его с сегментом, сценарием спроса, офером, каналом и сроком подготовки.

## v0.31

Gate 6 rebuilt as a quarterly campaign-decision calendar. Seasons are no longer the main containers. Q1–Q4 are used as the primary accordion containers; seasonal phase remains only as contextual guidance inside the quarter. Event cards were reduced to three working fields: Decision, Campaign, Preparation. Quarter accordion state is now keyed by unique quarterId, so Q1 / “Зима” can be closed normally and state does not break after saving.
